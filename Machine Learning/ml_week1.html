Foundation Concept

AI vs ML vs Deep Learning

These three terms get thrown around a lot — sometimes even interchangeably — but they're actually nested inside each other like Russian dolls. Let's unpack them once and for all.

🏛️
The Big Analogy

Think of it like this: AI is the idea of building a smart machine, ML is one powerful method to achieve that idea, and Deep Learning is an advanced sub-technique of ML. It's like cooking → recipes → molecular gastronomy.

Fig 1.1 — AI is the outer universe. ML lives inside it. Deep Learning is a powerful corner of ML.

🤖 Artificial Intelligence (AI)

AI is the broad goal: making machines smart. It includes anything from a simple if-else chess engine to a robot that folds laundry. It's the parent field.

📊 Machine Learning (ML)

ML is a specific approach to achieving AI — instead of hard-coding every rule, you give the machine data + examples and let it figure out the rules on its own. It learns from experience, just like you did when learning to ride a bike.

✅ Classic Definition

"Machine Learning is the field of study that gives computers the ability to learn without being explicitly programmed." — Arthur Samuel, 1959

🧠 Deep Learning (DL)

Deep Learning is ML that uses neural networks with many layers (hence "deep"). It's what powers face recognition, ChatGPT, and image generation. It shines when the data is huge and unstructured (images, text, audio).

Core Concept

Types of Machine Learning

There are three main flavours of ML. Think of them by how much help you give the machine while it's learning.

🏫
School Analogy

Supervised = Teacher gives you solved exam papers to study from. Unsupervised = Dump 1000 books on your desk and say "figure out what topics they're about". Reinforcement = You play a video game and get +10 points for good moves, -5 for bad ones.

Fig 2.1 — The three ML paradigms. Supervised is ~80% of real-world ML work.
Supervised
📚

Supervised Learning

You give the model input + correct output pairs. It learns to map inputs to outputs. Two sub-types: Classification (predicting a category — spam/not-spam) and Regression (predicting a number — house price).

Unsupervised
🔍

Unsupervised Learning

No labels. The model must discover structure on its own. Common tasks: Clustering (group similar customers), Dimensionality Reduction (shrink 100 features to 2 for visualization).

Reinforcement
🎮

Reinforcement Learning

An agent learns by interacting with an environment, receiving rewards for good actions and penalties for bad ones. It's used in robotics, games, and autonomous vehicles. Not covered much at Phase 1.

Python · Sklearn — Quick Taste of Supervised ML
# Supervised Learning in 10 lines — House Price Prediction
from sklearn.linear_model import LinearRegression
import numpy as np

# Training data: [size_sqft] → price
X_train = np.array([[600],[800],[1200],[1500],[2000]])   # input (features)
y_train = np.array([30,40,60,72,95])                    # output (labels) in lakhs

model = LinearRegression()
model.fit(X_train, y_train)           # LEARNING HAPPENS HERE 🧠

prediction = model.predict([[1000]])  # Predict price for 1000 sqft
print(f"Predicted price: ₹{prediction[0]:.1f} lakhs")
# Output: Predicted price: ₹49.5 lakhs

How Models Generalise

Instance-Based vs Model-Based Learning

Another way to classify ML: does the model memorise training examples, or does it build a mathematical model that generalises?

📖
Study Technique Analogy

Instance-based = learning by remembering exact past examples (cramming). Model-based = understanding the concept and deriving answers (deep learning strategy).

Fig 4.1 — Instance-based memorises data. Model-based finds the underlying pattern (the line).
AspectInstance-BasedModel-Based
How it learnsMemorises training examplesFits a mathematical model (line, curve, tree)
Prediction timeSlow (searches all data every time)Fast (just plug into a formula)
Memory usageKeeps all training data in RAMOnly stores model parameters
GeneralisationCan be too literalBetter at new unseen inputs

Practical Awareness

Challenges in Machine Learning

ML sounds magical, but it fails in very predictable ways. Knowing these failure modes is what separates a beginner from a practitioner.

Fig 5.1 — Three failure modes. The goal is always the middle — good generalisation.
📉
Underfitting (High Bias)
Model is too simple to capture the pattern. Like fitting a straight line to a curve. Fix: Use a more complex model or add more features.
📈
Overfitting (High Variance)
Model memorises training data including noise — fails on new data. Fix: Get more data, use regularisation, or simplify the model.
🗑️
Poor Quality Data
"Garbage in, garbage out." Missing values, outliers, duplicates, and wrong labels all hurt model quality. Chapters 8–12 of this course tackle this.
📦
Insufficient Data
ML models are data-hungry. Too little data → model can't learn meaningful patterns. Deep learning especially needs millions of examples.
🚫
Non-Representative Data
If training data doesn't represent real-world distribution, the model will be biased. Example: Training a loan-approval model only on urban applicants.
⚙️
Feature Engineering Gap
Choosing the wrong features — or missing key features — limits model ceiling. The right feature can 10× your model performance (Phase 3 of this course).
⚠️ Most Important Challenge: Overfitting

Overfitting is the number one reason ML models fail in production. A model might score 98% accuracy on training data and only 62% on real data. Always evaluate on a held-out test set the model has never seen. You'll learn cross-validation in Week 21.

Interview Prep

Week 1 — 10 Interview Questions

Click any question to reveal the answer. These are the most commonly asked Week 1 ML interview questions.

01
What is Machine Learning? How is it different from traditional programming?
▼

Machine Learning is a subset of AI where systems learn from data to make predictions or decisions — without being explicitly programmed with rules for every scenario.

Traditional Programming: Developer writes rules → Computer applies them → Output

Machine Learning: Data + Labels (Output) → ML Algorithm → Learns Rules → Can predict new outputs

Example: In traditional programming, a spam filter has hard-coded keywords like "lottery", "win", "click here". In ML, you train on thousands of emails labelled spam/not-spam, and the model discovers the patterns itself — including subtle ones a human might miss.

02
Explain the difference between AI, ML, and Deep Learning with an example.
▼

These are nested concepts — each is a subset of the one before:

  • AI (Artificial Intelligence) — The broad goal of making machines smart. Includes rule-based systems, search algorithms, robotics. Example: Google's chess engine (hard-coded rules).
  • ML (Machine Learning) — A technique to achieve AI by learning from data. Example: A spam classifier trained on email data.
  • Deep Learning — A powerful ML technique using multi-layer neural networks, needs large data + GPU. Example: ChatGPT, face recognition.

Key insight: Not all AI is ML, and not all ML is Deep Learning.

03
What are the three main types of Machine Learning? Give a real-world example of each.
▼
  • Supervised Learning: Model trains on labelled data (input + correct output). Example: Predicting house prices using past sale data (input = size, location; output = price). Used for classification and regression.
  • Unsupervised Learning: Model finds patterns in unlabelled data. Example: Grouping e-commerce customers by buying behaviour (nobody told the model what the groups are). Used for clustering and dimensionality reduction.
  • Reinforcement Learning: An agent learns by taking actions and receiving rewards/penalties. Example: AlphaGo learning to play chess by playing millions of games against itself.
04
What is the difference between Classification and Regression?
▼

Both are supervised learning tasks — the difference is what kind of output they predict:

  • Classification: Predicts a category (discrete output). Example: Is this email spam? → Yes / No. Is this tumour malignant? → Malignant / Benign. Algorithm examples: Logistic Regression, Decision Tree, SVM.
  • Regression: Predicts a number (continuous output). Example: What will this house sell for? → ₹45 lakhs. Algorithm examples: Linear Regression, Random Forest Regressor.

Quick trick: If the output is a label, it's classification. If it's a number, it's regression.

05
What is overfitting and underfitting? How do you detect and fix each?
▼

Overfitting: The model memorises the training data — it learns the noise, not the pattern. It scores very high on training data but poorly on test data. It "has no idea" what to do with unseen examples.

Detection: Train accuracy ≫ Test accuracy (large gap).

Fix: Get more training data, apply regularisation (L1/L2), use simpler model, dropout (DL), early stopping, cross-validation.

Underfitting: The model is too simple — it can't even learn from training data. Both train and test accuracy are low.

Detection: Both train and test accuracy are low.

Fix: Use a more powerful model, add more relevant features, reduce regularisation, train for longer.

06
What is the difference between Batch Learning and Online Learning?
▼
  • Batch Learning: The model is trained on the entire dataset at once, then deployed. Once deployed, it doesn't update — you have to re-train from scratch when new data comes. Good for stable, non-changing datasets. Example: Monthly retraining of a credit score model.
  • Online (Incremental) Learning: The model continuously updates as new data arrives. Uses partial_fit() in sklearn. Good for rapidly changing environments. Example: Stock price prediction, real-time feed ranking.

Risk of online learning: if bad/outlier data sneaks in, model quality degrades fast — you need data monitoring.

07
What is the difference between instance-based and model-based learning?
▼
  • Instance-based (Lazy Learning): The model stores all training examples and makes predictions by comparing new data to remembered examples. No explicit generalisation. Example: K-Nearest Neighbours (KNN) — finds the K most similar past examples and votes. Fast to train, slow to predict.
  • Model-based (Eager Learning): Builds an explicit mathematical model (like a line, tree, or decision boundary) from training data. Prediction is just plugging new data into the formula — very fast. Example: Linear Regression, Decision Trees.
08
What is the "No Free Lunch" theorem in Machine Learning?
▼

The No Free Lunch (NFL) theorem states that no single ML algorithm is universally the best for all problems. Every algorithm makes certain assumptions about the data — and those assumptions work great for some data distributions and poorly for others.

Practical implication: There's no "one algorithm to rule them all." You must try multiple algorithms and select the best one based on your specific data and problem. This is why data scientists run experiments with many models.

It's why sklearn pipelines, cross-validation, and model selection (Week 21) are so important — you can't just pick Random Forest by default and call it a day.

09
Name 5 real-world applications of each type of Machine Learning.
▼

Supervised Learning:

  • Email spam detection (classification)
  • House price prediction (regression)
  • Medical diagnosis — tumour malignant/benign (classification)
  • Student placement prediction (classification)
  • Credit default risk scoring (regression/classification)

Unsupervised Learning:

  • Customer segmentation in e-commerce (clustering)
  • Anomaly detection in bank transactions
  • Spotify music genre discovery
  • Document topic modelling (LDA)
  • Image compression using PCA

Reinforcement Learning:

  • AlphaGo / chess-playing AI
  • Self-driving car steering decisions
  • Robotic arm picking objects in warehouses
  • Algorithmic stock trading
  • RLHF — making ChatGPT responses more helpful
10
Why doesn't high training accuracy guarantee a good machine learning model?
▼

High training accuracy only measures how well the model memorised the examples it was shown — not how well it can generalise to new, unseen examples.

A model that achieves 100% training accuracy might be overfitting — it learned the training noise instead of the true underlying pattern. When deployed on real-world data, it may perform terribly.

This is why we always evaluate models on a held-out test set that the model never saw during training. Better yet, use k-Fold Cross-Validation (covered in Week 21) to get a robust, unbiased performance estimate.

Rule of thumb: The test set is the truth. Training set is just for learning.

  • Train accuracy: 99%, Test accuracy: 62% → Overfitting
  • Train accuracy: 65%, Test accuracy: 64% → Underfitting
  • Train accuracy: 92%, Test accuracy: 89% → Good model ✓
